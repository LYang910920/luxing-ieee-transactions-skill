## Attribution boundary

Use Tier A first/corresponding/original-draft papers for lexical and sentence-rhythm guidance. Tier B is secondary editorial evidence. Tier C informs research architecture and method/evidence combinations only unless new contribution evidence upgrades it.

Do not feed tutorial prose, README text, code comments, paper abstracts copied from publisher pages or collaborator-attributed sections into personal sentence-level calibration.

## Draft consumption

Apply safe traits:

- problem → formal object → verified computation → decision insight;
- abstract and Introduction move order;
- contribution dependency chain: model/problem → theory → algorithm → evidence;
- selective research-action verbs and transitions;
- measured result narration: observation → mechanism → bounded implication;
- sentence-length variation within the profile's soft ranges.

Check risky traits against journal and English norms:

- paragraph length and heading density;
- active/passive balance;
- acronym and named-framework density;
- promotional adjective strength;
- conclusion and limitations style;
- legacy grammar or connector repetition.

## Evidence-track addition

Before drafting an experiment section, select an evidence track and topology level. When real outcome data are unavailable, prefer a lawful semantically compatible observed topology with explicitly synthetic dynamics. Otherwise use disclosed statistic-matched or multi-family synthetic graphs. Never describe a real-topology/synthetic-dynamics study as real-world validation.

## Mandatory evaluator additions

The evaluator must run the claim gates and inspect:

- novelty evidence;
- theorem/implementation match;
- optimality/equilibrium verification;
- topology provenance, semantics and preprocessing;
- graph/data split leakage;
- simulator fidelity and independent checks;
- baseline fairness and repeated seeds;
- uncertainty, sensitivity and retained failures;
- identifiability and unilateral deviations where relevant;
- limitations and reality-status wording;
- no copyrighted article text, tutorial source or raw third-party data in release artefacts.
